# SPEC-INDEX

Core specifications define the Agent OS language, object model, and fundamental protocols. These are **not RFCs** — they are the stable foundation that all RFCs reference.

| ID | Title | Status | Version |
|----|-------|--------|---------|
| SPEC-0000 | Core Concepts & Object Model | Proposed | 1.0 |

## Process

- **SPECs** define what is. They are the authoritative reference.
- **RFCs** define how to implement. They reference SPECs.
- SPECs evolve via version bumps (1.0 → 1.1 → 2.0), not individual RFCs.
- SPEC changes require an ADR explaining the rationale.
